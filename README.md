# FT & CC Validator - 프롬프트 템플릿

> AI가 문서를 비교/분석하고, 문제 유형별로 정리해서 사용자 판단을 돕는 프롬프트 모음

---

## 📋 개요

| 구분 | 내용 |
|------|------|
| **AI 역할** | 분석/서술/정리 (판정 X) |
| **출력** | 문제 유형별 알람 형태 |
| **최종 판단** | 사용자가 직접 수행 |

```
[기존 방식] AI가 Pass/Fail 판정 → 사용자가 결과 확인
[변경 방식] AI가 차이점 분석 → 유형별 알람 → 사용자가 판단
```

## 🚀 사용 방법

### 1. 템플릿 복사

`prompts/user_template.md` 파일의 내용을 복사합니다.

### 2. 데이터 입력

- 기준 문서 (Reference/Master) 데이터 붙여넣기
- 샘플 문서 (검증 대상) 데이터 붙여넣기

### 3. AI에게 전달

GitHub Copilot Chat, Claude, ChatGPT 등에 전달:

```
prompts/ftcc_analyzer_prompt.md 를 참조하여 다음 두 문서를 비교 분석해줘:

[기준 문서]
...

[샘플 문서]
...
```

### 4. 결과 확인

알람 형식으로 정리된 분석 결과를 확인하고, 최종 판정을 수행합니다.

## 📊 분석 유형

| 아이콘 | 유형 | 심각도 | 설명 |
|--------|------|--------|------|
| 🔴 | 값 불일치 | 높음 | 기준값과 측정값이 다름 |
| 🟠 | 범위 이탈 | 높음 | 허용 오차 초과 |
| 🟡 | 누락 항목 | 중간 | 필수 항목 누락 |
| 🟡 | 미완료 항목 | 중간 | 빈 값, 서명 누락 등 |
| 🔵 | 형식 오류 | 낮음 | 데이터 형식 불일치 |
| ⚪ | 의심 항목 | 확인필요 | 추가 검토 필요 |

---

## 🎯 핵심 포인트

| 포인트 | 왜 중요한가 |
|--------|-------------|
| **판정 X, 분석 O** | 사용자가 최종 판단하도록 책임 명확화 |
| **유형별 분류** | 한눈에 문제 파악 가능 |
| **심각도 표시** | 우선순위 판단 용이 |
| **위치 명시** | 원본에서 바로 찾기 가능 |
| **차이 서술** | 무엇이 어떻게 다른지 이해 |

---

## 🔧 커스터마이징

### 프로젝트별 수정 포인트

1. **컬럼명 변경**: 실제 FT/CC 문서의 컬럼명으로 수정
2. **허용 범위 규칙**: 제품별 허용 오차 명시
3. **필수 항목 정의**: 반드시 있어야 하는 항목 목록
4. **형식 규칙**: 날짜, 숫자 등 데이터 형식 정의

## ⚠️ 주의사항

- AI 분석 결과는 **참고용**이며 최종 판정은 담당자가 수행
- 의심 항목은 **원본 문서와 대조** 확인 필요
- 복잡한 조건부 규칙은 **추가 프롬프트 설명** 필요
